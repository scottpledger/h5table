Can I load data into h5table using an array?

Yes, but only when you are initially creating it.

What properties are required on the objects passed in to the array?

The id and text properties are required on each object, and these are the properties that h5table uses for the internal data objects. Any additional paramters passed in with data objects will be included on the data objects that h5table exposes.

How should nested results be formatted?

How many levels of nesting are allowed?

Because h5table falls back to an <optgroup> when creating nested options, only a single level of nesting can be supported.

Why are <option> tags being created?

The data option is a shortcut that h5table provides which allows you to load options into your select from a data array.

My objects don't use id for their unique identifiers, what can I do?

You can re-map your identifier before passing it to h5table.
